C. VALUE

Source: Kano Model
Goal: Understand how features impact satisfaction

This is not about what they do.
This is about how satisfaction behaves.

The Kano Model Explained

The Kano model classifies attributes into types based on their impact on satisfaction.

Two axes:

X axis: Degree of fulfillment (feature not present → fully present)

Y axis: Customer satisfaction (dissatisfied → delighted)

There are three main categories:

1. Basic Needs (Must-be, Taken for granted)

If absent → strong dissatisfaction
If present → neutral, not exciting

Example:
Insurance must process claims correctly.
Patient expects doctor to be licensed.

No one praises basics.
But if missing, disaster.

2. Performance Needs (Competitive)

More performance → more satisfaction
Less performance → less satisfaction

Linear relationship.

Example:
Speed of approval
Time to appointment
Price
Coverage breadth

These are differentiators.

3. Exciters / Delighters

If absent → no dissatisfaction
If present → strong delight

Unexpected features.

Example:
Proactive case manager
Instant telehealth triage
AI-based symptom routing
Personalized health pathway

Over time:
Yesterday’s delighters become today’s basics.

This is critical.

Kano Question Design

You do not ask:
“Do you like this feature?”

You ask paired functional and dysfunctional questions:

If feature exists, how do you feel?
If feature does not exist, how do you feel?

Responses:

I like it

I expect it

I am neutral

I can tolerate it

I dislike it

Then you classify.
