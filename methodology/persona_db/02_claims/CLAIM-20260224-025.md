# Claim: Regulation requires payer FHIR APIs

## Statement
CMS requires impacted payers to implement and maintain HL7 FHIR APIs to improve electronic data exchange and streamline prior authorization.

## Evidence Basis
- CMS fact sheet explicitly states impacted payers are required to implement HL7 FHIR APIs and streamline prior auth. (Source: SOURCE-20260224-014)

## Reasoning
This is a hard constraint on insurer workflows and interoperability value requirements.
